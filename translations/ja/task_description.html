あなたは古き良き１２時間フォーマットを好んでいます。しかし、私達が暮らしている現代社会では２４時間フォーマットがより好んで使われますし、至るところで見かけます。このタスクは、次のルールに従って２４時間フォーマットの時刻を１２時間フォーマットに変換することです。 - 出力のフォーマットは、'hh:mm a.m.'（午前）もしくは、'hh:mm p.m.' (午後)
- 時刻が１０時以前の場合 - その時刻の十の位に'０'は付けずひと桁で表します。例えば：'9:05 a.m.'のようにです。
次のリンク先で12-hour formatいくつか有用な情報を見つけられます。

入力: ２４時間フォーマットの時刻(文字列型).

出力: １２時間フォーマットの時刻(文字列型).

例:

{% if interpreter.slug == "js-node" %} 
timeConverter('12:30') == '12:30 p.m.'
timeConverter('09:00') == '9:00 a.m.'
timeConverter('23:15') == '11:15 p.m.'
{% else %} 
time_converter('12:30') == '12:30 p.m.'
time_converter('09:00') == '9:00 a.m.'
time_converter('23:15') == '11:15 p.m.'
{% endif %}

どのように使われるか: 異なる時間フォーマットを扱う仕事をする際に役立ちます。

事前条件:
'00:00' <= time <= '23:59'

Also interesting fact about "12:00 p.m." and "12:00 a.m.". The name of this fact is "Confusion at noon and midnight" - problem using "p.m./ a.m." with this magical time "12:00". You can find information about this in Wikipedia or just google it.
